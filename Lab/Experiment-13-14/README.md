# Experiment-13&14 — File Handling Using Java I/O Streams + Generics

This lab sheet combines **Experiment-13** and **Experiment-14** as per the lab syllabus.

## General instructions (lab manual)

- Write and execute programs independently.
- Follow standard Java coding conventions.
- Submission must include: **source code**, **output**, and brief **observations**.

## Experiment 13 (Syllabus)

**Objective:** To perform file operations using Java I/O streams.  
**Problem Statement:** Develop programs using FileReader and FileWriter.  
**Expected Outcome:** Students will handle persistent data.

## Experiment 14 (Syllabus)

**Objective:** To implement generic classes and methods.  
**Problem Statement:** Design programs using generics.  
**Expected Outcome:** Students will understand type safety.

## Lab tasks (recommended scenario)

### Task A — File I/O (Exp-13)

Create `FileIOStreamsDemo.java`:

1) Write 3 lines to `input.txt` using `FileWriter`.
2) Read and print the file contents using `FileReader`.
3) Copy `input.txt` to `copy.txt` using a character buffer.
4) Print how many characters were copied.
5) Use `try-with-resources` and handle `IOException`.

### Task B — Generics basics (Exp-14)

Create `GenericsDemo.java`:

1) Create a generic class `Box<T>` that stores a single value and returns it via `get()`.
2) Create a generic method `printTwice(T x)` and call it with different types.
3) Demonstrate:
- `Box<Integer>`
- `Box<String>`

Optional:
- create a generic `Pair<K,V>` and print it.

## Deliverables

- `FileIOStreamsDemo.java`
- `GenericsDemo.java`
- `input.txt` and `copy.txt` (generated by program)
- Sample console output for both programs
- Observations (5–10 lines): what try-with-resources does; how generics reduce casting and `ClassCastException`

## Optional extensions

- Use `BufferedReader`/`BufferedWriter` and compare performance for larger files.
- Add a method `swap(Pair<K,V>)` and discuss why it is (or isn’t) type-safe.
