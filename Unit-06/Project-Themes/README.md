# Unit 06 Project Themes

This folder contains the approved project-theme options for `Unit 06`.

Do not start from this folder first.

## Read This Folder After

You should reach this folder only after reading the shared Unit 06 guidelines in:

- `../Student-Guidelines/README.md`
- `../Student-Guidelines/unit-06-student-handbook.pdf`

Your group should also have:

- formed the group correctly
- selected the group leader
- requested teacher approval
- received project approval

## How To Use This Folder

Follow this order.

1. Read the approved project folder name carefully.
   - Make sure you open the folder for the theme approved for your group.
2. Read `project-description.pdf`.
   - This gives the overall scope of the project.
3. Read `project-phases.pdf`.
   - This shows how the project should progress phase by phase.
4. Read the files inside `phase-hints/` in order:
   - `phase-0-proposal-and-backlog.pdf`
   - `phase-1-requirements-and-use-cases.pdf`
   - `phase-2-uml-and-package-design.pdf`
   - `phase-3-database-and-dao-layer.pdf`
   - `phase-4-ui-and-integration.pdf`
   - `phase-5-testing-build-and-demo.pdf`

## What Each Project Folder Gives You

Each project folder gives:

- one approved project theme
- one project description
- one phase plan
- one hint file for each major phase of execution

Use these files as execution guidance, not as a substitute for your own design work.

## Project Selection Rule

Your group should work only on the project theme approved by the teacher.

Do not:

- switch the project theme without approval
- mix multiple project themes into one submission
- create a repository before teacher approval

## Recommended Working Pattern

After your theme is approved:

1. open the approved project folder
2. read `project-description.pdf`
3. finalize your scope
4. follow `project-phases.pdf`
5. use the phase-hint PDFs while preparing each deliverable
